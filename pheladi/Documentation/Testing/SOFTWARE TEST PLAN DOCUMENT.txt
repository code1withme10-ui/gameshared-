SOFTWARE TEST PLAN DOCUMENT
Creche Management Website System


1.Introduction

1.1 Purpose

The purpose of this Test Plan is to define the testing approach, resources, test cases, and procedures used to verify that the Crèche Website System meets all functional and non-functional requirements. The test plan ensures that the system operates correctly before deployment and that all defects are identified and resolved.

1.2 Scope

Testing will verify the functionality and reliability of the Crèche Website which is developed. The following modules will be tested:
Home Page
About Page
Gallery
Admission Application Form
File Upload System
Login and Authentication
Headmaster Dashboard
JSON Data Storage
Security and Performance

2. Test Objectives
The objectives of the testing process are to:
Verify that the system meets all functional requirements.
Validate user input fields and form submissions.
Ensure secure login and access control.
Confirm correct data storage using JSON.
Detect and correct system defects before deployment.
Ensure system performance and reliability.

3. Test Scope
3.1 In Scope
The following components will be tested:
Home Page
About Page
Gallery Page
Admission Application Form
File Upload System
Login System
Headmaster Dashboard
JSON Data Handling
Security Features
Performance

3.2 Out of Scope
The following will not be tested in this phase:
Third-party email/SMS services
External hosting infrastructure
Future modules such as payments and parent portals

3.3 Types of Testing
Unit Testing
Objective:
To test individual components independently.

Performed By:
Developer

Procedure:
Test each form field validation.
Test login authentication function.
Test file upload function.
Test status update logic.
Test notification sending function.
Expected Result:
Each function works correctly in isolation.
=======================================================================================
Integration Testing
Objective:
To verify that system components work together.

Procedure:
Submit admission form.
Check if data is saved in database.
Login as Headmaster.
Verify application appears on dashboard.
Change status to “Accepted”.
Confirm notification is sent.

Expected Result:
All modules communicate correctly.
======================================================================================
System Testing
Objective:
To test the complete system.

Procedure:
Navigate all website pages.
Test full admission process.
Test login for all roles.
Test dashboard reporting.
Test exporting PDF/Excel.
Expected Result:
Entire system works as specified in SRD.
================================================================================
User Acceptance Testing (UAT)
Objective:
To confirm system meets stakeholder expectations.

Performed By:
Crèche Management / Headmaster

Procedure:
Headmaster logs in.
Reviews applications.
Accepts/declines application.
Reviews reports.
Confirms system usability.
Expected Result:
Stakeholders approve system for deployment.
=============================================================================
Security Testing
Procedure:
Attempt SQL injection in login.
Attempt to access dashboard without login.
Attempt to change URL to access restricted pages.
Check password storage in database.
Check HTTPS encryption.
Expected Result:
System blocks all unauthorised access attempts.
==============================================================================
Performance Testing
Procedure:
Load home page.
Measure load time.
Simulate multiple users submitting forms.
Test dashboard response time.
Expected Result:
Page loads under 3 seconds.
==================================================================================
Compatibility Testing
Test On:
Google Chrome
Microsoft Edge
Mozilla Firefox
Mobile phone
Tablet
Expected Result:
Layout responsive and functional across devices.
=======================================================================================
Accessibility Testing
Procedure:
Check font readability.
Test colour contrast.
Test keyboard navigation.
Add alt text to images.
Expected Result:
Accessible for users with visual impairments.
