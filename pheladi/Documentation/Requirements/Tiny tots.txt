INTRODUCTION
=======================
The purpose of this Software Requirements Document (SRD) is to describe the functional, non-functional, data, and validation requirements for the development of a web-based crèche management and information system. This document will be used by developers, stakeholders, and testers to understand the system features and guide the design, development, and testing processes.

PURPOSE
======================
The purpose of this Software Requirements Document (SRD) is to provide a detailed description of the functional, non-functional, data, and validation requirements for the crèche website system. This document serves as a communication tool between stakeholders, developers, and testers to ensure a shared understanding of the system’s features and objectives. It will guide the design, development, testing, and implementation of the system, ensuring that it meets the needs of parents, staff, and administrators.

4.2 Headmaster Requirements
--------------------------------------------------------------------------
FR-HMA 01 Authentication and Authorisation
The system shall require secure login credentials for access to the Headmaster Page.
The system shall restrict access to authorised users with the Headmaster role only.
The system shall log the login and logout activities of the headmaster.

FR-HMA 02 Dashboard Overview
The system shall display a dashboard summary upon login.
The dashboard shall show:
Total number of applications
Number of pending applications
Number of accepted applications
Number of declined applications
The system shall notify the headmaster of new applications.

FR-HMA 03 View Admission Applications
The system shall display a list of all admission applications.
Each application record shall display:
Child’s full name
Parent/guardian name
Application date
Current application status
The system shall allow filtering applications by status (Pending, Accepted, Declined).
The system shall allow sorting applications by date.

FR-HMA 04 View Application Details
The system shall allow the headmaster to view full application details.
The application details shall include:
Child information (name, date of birth, age, gender)
Parent/guardian information (name, contact number, email address, physical address)
Uploaded supporting documents
Preferred start date
Additional notes provided by the parent/guardian

FR-HMA 05 Review and Decision Management
The system shall allow the headmaster to review applications individually.
The system shall provide the following actions:
Accept application
Decline application
Keep application pending
Upon acceptance, the system shall update the application status to “Accepted”.
Upon decline, the system shall update the application status to “Declined”.
The system shall allow the headmaster to capture optional notes or reasons for decisions.

FR-HMA 06 Notifications
The system shall automatically notify parents/guardians via email or SMS when an application status changes.
The notification shall include:
	•	Application status
	•	Crèche contact information
	•	Optional message from the headmaster

FR-HMA 07 Application History and Audit Trail
The system shall maintain a history of all application status changes.
The audit trail shall record:
	•	Date and time of decision
	•	User who made the decision
	•	Previous and new status

FR-HMA 08 Search Functionality
The system shall allow searching applications by:
	•	Child’s name
	•	Parent/guardian name
	•	Application reference number

FR-HMA 09 Data Management and Reporting
The system shall allow the headmaster to view a list of accepted learners.
The system shall allow exporting application data in PDF or Excel format.
The system shall allow archiving of closed or old applications.

5.Login Requirements
------------------------------------------------------------------------
FR-LOG 01 Login Interface
The system shall provide a login page accessible via the crèche website.
The login page shall include the following input fields:
	•	Username or email address
	•	Password
	•	The login page shall include a login button to submit credentials.

FR-LOG 02 Authentication
The system shall validate user credentials against stored records.
The system shall deny access for invalid credentials.
The system shall display appropriate error messages for failed login attempts.
The system shall support secure password storage (hashed passwords).

FR-LOG 03 Role-Based Access Control
The system shall assign a role to each registered user.
The system shall determine user permissions based on their assigned role.
Upon successful login, the system shall redirect users according to their role:
	•	Headmaster → Headmaster Dashboard
	•	Staff → Staff Dashboard
	•	Administrator → Admin Dashboard

FR-LOG 04 Session Management
The system shall create a secure session upon successful login.
The system shall automatically log out users after a period of inactivity.
The system shall provide a logout option on all secured pages.

FR-LOG 05 Access Restriction
The system shall prevent users from accessing pages not authorised for their role.
The system shall redirect unauthorised access attempts to an error or login page.

FR-LOG 06 Password Management
The system shall provide a “Forgot Password” functionality.
The system shall allow users to reset their password via a secure process.
The system shall enforce password complexity rules.

5.7 Login Audit and Monitoring
The system shall record login attempts.
The system shall log the following details:
	•	User ID
	•	Date and time of login
	•	Login success or failure

2. Organisation Background
------------------------------------------
Tiny Tots Creche in Musina began as a locally focused early childhood development centre dedicated to nurturing and educating young children in the community. Official records indicate that Tiny Tots was registered in 2014, marking its formal establishment as a licensed daycare and preschool provider in the region.

Since its founding, the creche has grown into a trusted place for families in Musina, offering age-appropriate learning and play activities for children from infancy through the early years. Staff at Tiny Tots focus on fostering cognitive, social, emotional, and physical development through guided play, exploration, and interactive learning — helping prepare children for the transition into formal schooling.

Over the years, Tiny Tots has earned a positive reputation among parents and caregivers for its warm environment and committed team. Though small, the centre has been regarded locally as one of the respected early childhood education options in the town.

VALIDATION REQUIREMENTS
-----------------------------------------------------
Validation:
Allowed formats: PDF, JPG, PNG
Max size: 2MB per file
Multiple upload attempts allowed
The system must show a preview if the document is an image
The system must reject invalid file types
The system must show error messages for failed uploads
All required fields must be completed
Email must include “@” and valid domain
Phone number must be numeric and properly formatted
Date of birth cannot be a future date
Grade must be selected
Check file type before upload
Check file size before upload
Show real-time error messages

Submission Requirements:
Upon submitting the form:
All fields must be validated.
All documents must be uploaded.
The application will only save if the validation passes.
Application status defaults to "pending".
A success message must be displayed after submission, and the  parent must view a summary of the submission.
