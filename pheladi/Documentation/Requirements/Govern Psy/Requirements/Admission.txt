 Introduction

Purpose
This document defines the complete requirements for the Parent Child Application Form, where parents enter full details about their child and upload supporting documents. The system stores all application information in a JSON file instead of a database. The system must be developed using html and php.

Scope
The system must allow the parents to:
Enter child and parent details.
Upload required supporting documents.
Submit an application.
The system will not use a database to store files but will use JSON for storage.

Functional Requirements
1.1 Child Information required fields:
Child Full Name
Date of Birth
Gender
Grade Applying For
Residential Address
Disability(Y/N)
Medical allegies

Grade Categories
Parents must select from the following options:
Category	Age Range
Toddlers	2 – 3 years
Playgroup	3 – 4 years
Pre-School	4 – 5 years
Foundation Phase 6 - 9 years

Selection Method
The field must be a dropdown menu or radio buttons.
The system must validate that the child’s age matches the selected category.

Default Behavior
Dropdown should display “Select Category” by default.
Other categories should be disabled if the age does not match.

1.2 Parent Information required fields:
Parent Full Name
ID No/Passport No
Relationship to Child
Email Address
Phone Number
Residential Address

1.3 Emergency contact(Other than parent/guardian)
Full Name
Residential address
Phone number

Validation:
Email must be a valid format.
Phone number must match a valid number pattern.

1.4 Supporting Documents Upload:
Parents must upload the following:
Birth Certificate
Clinical report
Parent/Guardian ID copy
A copy of previous school report


Validation:
Allowed formats: PDF, JPG, PNG
Max size: 2MB per file
Multiple upload attempts allowed
The system must show a preview if the document is an image
The system must reject invalid file types
The system must show error messages for failed uploads

1.5 Submission Requirements:
Upon submitting the form:
All fields must be validated.
All documents must be uploaded.
The application will only save if the validation passes.
Application status defaults to "pending".
A success message must be displayed after submission, and the  parent must view a summary of the submission.

1.6 JSON File Handling
New applications must not overwrite existing applications.
New object must be appended per application.
JSON file must remain well-structured and readable.

1.7 Validation Requirements
All required fields must be completed
Email must include “@” and valid domain
Phone number must be numeric and properly formatted
Date of birth cannot be a future date
Grade must be selected
Check file type before upload
Check file size before upload
Show real-time error messages
