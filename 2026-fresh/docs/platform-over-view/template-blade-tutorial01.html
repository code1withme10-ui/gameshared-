Using a Common Layout with BladeOne

Structure for layout.blade.php and page views

1. The Concept

A layout file defines the shared structure of your pages:

  • HTML head section
  • Navigation
  • Footer
  • Common scripts and styles

Individual pages only define the content that changes.

2. Suggested Folder Structure

resources/
└── views/
    ├── layouts/
    │   └── app.blade.php
    │
    ├── learners/
    │   ├── index.blade.php
    │   └── create.blade.php
    │
    └── dashboard.blade.php

Keep layouts separate inside a layouts folder.

3. Example: layouts/app.blade.php

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>@yield('title', 'Crèche Platform')</title>

    <link rel="stylesheet"
          href="https://www.w3schools.com/w3css/4/w3.css">

    @yield('styles')
</head>
<body class="w3-light-grey">

    <!-- Navigation -->
    <div class="w3-bar w3-blue">
        <a href="/" class="w3-bar-item w3-button">Home</a>
        <a href="/learners" class="w3-bar-item w3-button">Learners</a>
        <a href="/settings" class="w3-bar-item w3-button">Settings</a>
    </div>

    <div class="w3-content w3-padding-32" style="max-width:1100px">
        @yield('content')
    </div>

    <footer class="w3-container w3-blue w3-center w3-padding-16">
        Multi-Tenant Crèche Platform
    </footer>

    @yield('scripts')
</body>
</html>

The key concept is @yield(). It defines placeholders that child pages will fill.

4. Example: learners/create.blade.php

@extends('layouts.app')

@section('title', 'Register Learner')

@section('content')
    <div class="w3-card w3-white w3-padding">
        <h2>Register Learner</h2>

        <form method="POST" action="/learners">
            <p>
                <label>First Name</label>
                <input class="w3-input w3-border"
                       type="text"
                       name="first_name"
                       required>
            </p>

            <p>
                <label>Date of Birth</label>
                <input class="w3-input w3-border"
                       type="date"
                       name="date_of_birth"
                       required>
            </p>

            <button class="w3-button w3-blue">
                Save
            </button>
        </form>
    </div>
@endsection

@extends() tells BladeOne to use the layout. @section() fills the layout’s placeholders.

5. Optional: Adding Page-Specific Scripts

@section('scripts')
<script>
    console.log("Learner page loaded");
</script>
@endsection

This script will be injected into the @yield('scripts') area of the layout.

6. Rendering the View (Controller)

return $this->view('learners.create', [
    'tenant' => $tenant,
]);

BladeOne automatically resolves dot notation to folders: learners.create → learners/create.blade.php

7. Summary

  • layouts/app.blade.php → Shared structure
  • @yield() → Placeholder
  • @extends() → Use layout
  • @section() → Fill placeholder

This structure keeps your UI consistent, clean, and easy to maintain as your platform grows.
