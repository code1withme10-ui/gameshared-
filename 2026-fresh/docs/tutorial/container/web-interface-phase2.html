Phase 2 – Make Sender Functional

Convert your HTML page into a working PHP message sender

Objective

You will now take the static sender.php and make it functional:

  • Use a POST form to send messages
  • Execute the CLI command php index.php <type> <message>
  • Capture and display the output directly on the page

This is the first step toward using PHP and AI together to control message sending.

Step 1: Verify Your Workspace

Make sure you are in the correct folder. Your files should live here:

cd /var/www/tutorial/containers

The server will be started from the public/ folder.

Step 2: Start PHP Server

Open a terminal and start the built-in PHP server:

cd /var/www/tutorial/containers/public
php -S localhost:8000

Open your browser at http://localhost:8000/sender.php.

Step 3: Convert HTML to Functional PHP

Ask the AI to update sender.php with the following instructions:

"Take the static sender.php HTML page. Convert it to a PHP page that handles form POST. The form has a select for type (SMS, Email, WhatsApp) and a text area for the message. When the form is submitted, execute php index.php <type> <message> using PHP shell_exec() and display the output below the form. Keep it simple, no frameworks, no CSS changes."

Copy the AI output into sender.php in your workspace.

Step 4: Test Your Page

  1. Ensure index.php exists in the same folder and works with CLI commands.
  2. Open sender.php in your browser: http://localhost:8000/sender.php
  3. Select a message type and enter a test message.
  4. Click "Send Message".
  5. The result of the CLI execution should appear below the form.
If the output appears, your sender page is functional and ready for the next phase.

Step 5: Reflection

You have now converted static HTML to a working PHP page that interacts with your CLI-based message system. In the next phase, we will explore handling multiple message channels and improving the UI experience.
