Phase 3 – Multi-Channel Sender (Advanced)

Optional advanced exercise: extend your sender for multiple channels

Objective

Extend your existing sender.php to handle multiple message channels:

  • Email (already functional)
  • SMS
  • WhatsApp

This exercise demonstrates **flexibility and loose coupling** using the same CLI-based architecture. You will also practice updating forms and processing inputs dynamically.

Step 1: Verify Workspace

Make sure your workspace is correct. Files should live here:

cd /var/www/tutorial/containers

Your public server folder remains public/ for sender.php.

Step 2: Start PHP Server

cd /var/www/tutorial/containers/public
php -S localhost:8000

Open http://localhost:8000/sender.php in your browser.

Step 3: Update Sender via AI Prompt

Ask the AI to update sender.php with the following instructions:

"Enhance the existing sender.php page. The form should allow selecting a message type: Email, SMS, or WhatsApp. When submitted, execute the CLI command php index.php <type> <message> using PHP shell_exec(). Display the output below the form. Keep the page simple: no frameworks, no JavaScript, no CSS changes. Only extend functionality for multiple channels."

Copy the AI output into sender.php and save.

Step 4: Test Your Multi-Channel Sender

  1. Ensure index.php supports all channels: Email, SMS, WhatsApp.
  2. Open sender.php in the browser.
  3. Select each channel one by one and enter a test message.
  4. Click "Send Message".
  5. Verify that output shows the correct CLI execution result for each channel.
Successfully sending via multiple channels shows your page is flexible and ready for future extensions.

Step 5: Reflection

Compare this advanced sender with Phase 2:

  • You reused the same form structure
  • You dynamically handled multiple message types without changing core logic
  • You reinforced the concept of loose coupling

Optional: consider how this architecture could be extended for Telegram or other channels with minimal changes.
