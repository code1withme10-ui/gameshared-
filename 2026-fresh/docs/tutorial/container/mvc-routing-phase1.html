Phase 1 – Manual Routing Inside index.php

Objective

In Phase T, we centralized web access through public/index.php.

In this phase, we will introduce simple manual routing logic.

We are still NOT creating a Router class.

What Must Change / Key Concepts

index.php must now:

  • Detect the current URI
  • Load different logic based on that URI

Example logic:

$uri = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH); if ($uri === '/messages/create') { require 'sender.php'; } elseif ($uri === '/messages') { echo "Listing messages..."; } else { echo "Page not found."; }
This is manual routing. No abstraction yet.

AI Prompt / Guidance for Learner

Before modifying index.php, paste its full contents into AI.

Then instruct AI:

This is Phase 1 – Manual Routing. Please help me modify my public/index.php to: - Detect the URI - Load sender.php when URI is /messages/create - Display simple text for /messages - Show "Page not found" for unknown routes - Without introducing Router class - Without introducing Controller

Ensure no existing functionality breaks.

What to Check / Verification Steps

  1. Start server:
php -S localhost:8000 -t /var/www/tutorial/containers/public/
  1. Visit:
http://localhost:8000/messages/create
  1. Confirm sender form appears.
  1. Visit:
http://localhost:8000/messages
  1. Confirm text "Listing messages..." appears.
  1. Visit unknown route:
http://localhost:8000/unknown
  1. Confirm "Page not found." appears.
If all checks pass, manual routing is working.

Terminology

  • Route – A URL mapped to logic.
  • Dispatching – Deciding what code runs based on the URL.
  • Conditional Routing – Using if/else to decide behavior.

Reflection

If we add 10 more routes, what happens to index.php?
