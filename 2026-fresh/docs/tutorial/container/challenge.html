Challenge Phase

Prove that your architecture is flexible.

Your Objective

Add a completely new message channel.

Examples:

  • TelegramNotifier
  • PushNotifier
  • SlackNotifier

You must decide how to prompt the AI.

Rules

  • Do not modify NotificationService
  • The new class must implement NotifierInterface
  • Only update Container.php to support the new type
  • Keep everything CLI-based
  • Keep code simple

Run From Command Line

After implementing your new channel:

php index.php telegram "Hello from Telegram"

Replace telegram with your chosen type.

Self-Check

  • Did you create only one new file?
  • Did you avoid changing NotificationService?
  • Did you update only the container logic?
  • Does email still work?
  • Does SMS still work?
If you modified multiple core files, your system may still be tightly coupled.

Terminology

Scalability: The ability to grow without rewriting core logic.

Open for Extension: You can add new behavior without modifying existing stable code.
