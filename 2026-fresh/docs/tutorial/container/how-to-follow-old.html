How to Follow This Tutorial

Step-by-step professional workflow using AI as your tool

Step 1: Prepare Your Environment

  • Create a new folder: /var/www/tutorial/containers/
  • Do NOT use your existing work sandbox
  • Place all tutorial files inside this folder
This tutorial must be completely isolated from production or client work.

Step 2: Start a Fresh AI Session

  • Log out of any existing AI chat sessions
  • Start a brand new session
  • Do NOT attach the chat to any previous project
Fresh context ensures AI guidance matches the phase instructions exactly.

Step 3: Initialize AI with Prompt and SOP

  1. Open prompt.md and copy its contents
  2. Paste into the new AI chat and send
  3. Open SOP.md, copy all content, paste into the same chat, and send
The SOP ensures AI stays within tutorial constraints. Professional developers control their tools.

Step 4: Follow Phases Strictly

This tutorial is divided into phases. Complete each phase before moving to the next.

  • Phase 1 – Hard-coded implementation (feel the pain)
  • Phase 2 – Introduce an interface
  • Phase 3 – Introduce a container
  • Phase 4 – Extend with new message channels
Skipping phases will prevent you from understanding the architecture mindset.

Phase 1 – Hard-Coded Implementation

In this phase, you will feel the pain of tight coupling. You will manually instantiate the EmailNotifier and NotificationService in index.php.

Instructions:

  1. Open index.php
  2. Require EmailNotifier.php and NotificationService.php
  3. Manually create objects: 
    $notifier = new EmailNotifier();
$service  = new NotificationService($notifier);
$service->send($message);
  4. Run the CLI command to test: 
    php index.php email "Hello World"

Sample AI Prompt:

We are in Phase 1. I want a hard-coded version of the notification system. Do not introduce interfaces or containers. Use simple PHP classes. CLI only.

Step 5: Work With AI Professionally

Do not ask vague questions like:

“Build me a notification system.”

Instead, be precise:

“We are in Phase 1. Hard-coded version only. Do not introduce interfaces yet.”

If AI introduces frameworks, namespaces, or advanced patterns, remind it of the SOP constraints.

Professional Conduct & Motivation

  • Write code yourself
  • Read every line generated by AI
  • Understand before copying
  • Refactor manually when instructed

You are not here to generate code. You are here to understand architecture and learn to use AI as a professional tool.

Professionals control tools. Tools do not control professionals.
