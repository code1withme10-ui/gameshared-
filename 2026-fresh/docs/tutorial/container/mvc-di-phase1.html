Dependency Injection – Phase 1

Step 1: Modify MessageController

Update the constructor to accept MessageService as a dependency:

1. Open MessageController.php. 2. Add a constructor that takes MessageService: function __construct($messageService) { $this->messageService = $messageService; // Save for use in controller methods } 3. Remove any new MessageService() inside the controller. 4. All controller methods should use $this->messageService instead of creating a new instance.

Step 2: Verification

  • Controller no longer contains new MessageService()
  • Constructor accepts $messageService
  • All methods call $this->messageService
  • Application behavior should remain unchanged
These changes only restructure your code for DI; output should remain the same.

Step 3: Workspace Commands

Make sure you are in the tutorial folder and PHP server is running:

cd /var/www/tutorial/containers php -S localhost:8000 -t public/

Navigate to:

http://localhost:8000/messages

Step 4: Reflection

  • Notice how the controller no longer creates its own dependencies
  • Dependencies are now passed in from the outside (preparing for easier testing)
  • Compare this with the Container lesson – similar principle but manual
Manual DI sets the foundation for more flexible, decoupled architecture.
