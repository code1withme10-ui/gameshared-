Phase 2 – Introduce Interfaces

Learn to instruct AI to refactor code toward abstraction.

Objective

In Phase 1, your service depended directly on EmailNotifier. Now you will guide AI to introduce an abstraction so the service depends on a contract instead of a concrete class.

Your goal is not to copy code. Your goal is to instruct AI correctly.

What Must Change

  • Create a NotifierInterface
  • Make EmailNotifier implement that interface
  • Modify NotificationService to depend on the interface
  • Do NOT introduce containers yet

Your AI Prompt

We are in Phase 2 of a learning tutorial. Refactor the existing notification system to introduce a NotifierInterface. NotificationService must depend on the interface instead of EmailNotifier. Keep everything simple. No containers. No frameworks. CLI only. Pure PHP. No namespaces.

What To Check Carefully

After AI generates the code, do the following:

  1. Open NotifierInterface.php. Does it contain a method called send?
  2. Open EmailNotifier.php. Does the class say implements NotifierInterface?
  3. Open NotificationService.php. In the constructor, do you see NotifierInterface instead of EmailNotifier?
  4. Run the system again. Does it still send the message successfully?
If the system still works and EmailNotifier is no longer directly required inside NotificationService, you have successfully reduced coupling.

Terminology

Interface:A contract that defines what methods must exist, but not how they work.

Abstraction:Hiding implementation details behind a simple contract.

Reflection

If you wanted to add SMS next, would you need to change NotificationService? If yes, review your implementation again.
