Architectural Training: Containers & Interfaces

Phase 1 – Hard-Coded Implementation

Welcome

This tutorial is designed to strengthen your architectural thinking. You are not building a feature for a client. You are building professional discipline.

The goal is to understand:

  • What tight coupling feels like
  • Why abstraction matters
  • How professional developers structure systems
  • How centralized object creation can improve maintainability

Important Mindset

This tutorial must be completed in a new folder:

/var/www/tutorial/containers/

This folder is your architectural training space — separate from normal work.

What You Will Build

You will create a simple Notification System that can send messages via:

  • Email

At first, the system will be tightly coupled and rigid. You will experience the limitations.

Later, step by step, it will become flexible, extendable, and professional.

Technology Rules

  • Pure PHP (8.1+)
  • No frameworks
  • No Composer
  • No namespaces
  • No autoloaders
  • Simple require statements only
  • CLI execution only
  • Keep code small, readable, and well-commented

Expected Outcome

By the end of Phase 1, you should be able to explain:

  • What tight coupling is
  • How dependency affects flexibility
  • Why object creation matters

If you can explain these concepts clearly without looking at the code, you are thinking like a professional developer.

Architectural discipline creates scalable systems.
