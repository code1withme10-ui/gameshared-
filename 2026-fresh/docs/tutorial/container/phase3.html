Phase 3 – Introduce a Container

Stop manually creating objects. Centralize object creation.

Objective

In previous phases, you manually created objects in index.php. Now you will introduce a simple Container class responsible for creating and returning objects.

The goal is to move object creation to one central place.

What Must Change

  • Create a new file: Container.php
  • The container must create EmailNotifier
  • The container must create NotificationService
  • index.php must stop using new EmailNotifier() directly

Your AI Prompt

We are in Phase 3 of a learning tutorial. Create a simple Container class that is responsible for creating EmailNotifier and NotificationService. index.php should no longer manually create these objects. Keep everything simple. No frameworks. No namespaces. No advanced patterns. CLI only.

What To Check Carefully

  1. Open Container.php. Does it create and return a NotificationService?
  2. Open index.php. Do you still see new EmailNotifier() there? If yes, the refactor is incomplete.
  3. Does index.php now use the Container to get the service?
  4. Run the system again. Does it still send the message?
If index.php is simpler and object creation happens inside Container.php, you have centralized object creation successfully.

Terminology

Container: A central place responsible for creating and providing objects.

Dependency Injection: Giving an object the dependencies it needs instead of creating them inside.

Reflection

If you wanted to change from EmailNotifier to SmsNotifier, how many files would you need to modify now?
