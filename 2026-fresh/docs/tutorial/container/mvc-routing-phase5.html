Phase 5 – Handling CLI Message Sending & Status

Objective

Extend the Controller to send messages using your CLI script and store the status. Users will see whether a message was sent successfully.

Controller calls CLI using shell_exec and passes type + message.

What Must Change / Key Concepts

  • Update MessageController::store() to execute CLI script:
    • $command = "php ../index.php $type \"$message\"";
    $output = shell_exec($command);
  • Store $output in MessageModel alongside type, message, timestamp
  • Update views to display Status in list and detail pages
  • Router remains unchanged
CLI integration simulates real-world message sending in a simple way.

AI Prompt / Guidance for Learner

Paste full current index.php, Router.php, Controller.php, MessageController.php, MessageModel.php into AI first.

Then instruct AI:

Phase 5 – CLI Message Sending & Status 1. Update MessageController::store(): - Capture $_POST['type'] and $_POST['message'] - Execute CLI: $output = shell_exec("php ../index.php $type \"$message\"") - Save message via MessageModel::create([type, message, status, created_at]) 2. Ensure index() and show() display status 3. Preserve all previous functionality 4. Keep code commented and minimal

What to Check / Verification Steps

  1. Start server:
php -S localhost:8000 -t /var/www/tutorial/containers/public/
  1. Visit message form:
http://localhost:8000/messages/create
  1. Submit a message via form (SMS / Email / WhatsApp)
  1. Visit message list:
http://localhost:8000/messages
  1. Confirm status column shows CLI output
  1. Click a message to view details:
http://localhost:8000/messages/show?id=0
If all messages show their CLI status, phase integration is successful.

Terminology

  • CLI – Command Line Interface
  • shell_exec() – Executes shell commands from PHP
  • Status – Result returned from CLI script

Reflection

How does integrating CLI message sending separate concerns and prepare for future message channels?

Controller handles requests, Model handles data, CLI handles sending – clear separation.
