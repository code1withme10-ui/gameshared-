Phase 4 – Extend with a New Channel

Experience the power of interfaces and containers.

Objective

You will now add a new message channel (e.g., SmsNotifier or WhatsAppNotifier).

The goal is to observe how little code needs to change.

What Must Change

  • Create a new notifier class (e.g., SmsNotifier)
  • Make it implement NotifierInterface
  • Update Container.php to return the correct notifier based on input
  • Do NOT modify NotificationService

Your AI Prompt

We are in Phase 4 of a learning tutorial. Add a new notifier called SmsNotifier that implements NotifierInterface. Update the Container so it can return either EmailNotifier or SmsNotifier depending on the send type. Do not modify NotificationService. Keep everything simple. No frameworks. No namespaces. CLI only.

What To Check Carefully

  1. Does SmsNotifier implement NotifierInterface?
  2. Did you avoid modifying NotificationService?
  3. Is the selection logic inside Container.php?
  4. Does the system still work for email?
  5. Does it now work for SMS?
If you only modified or added the notifier and container, your architecture is flexible.

Run From Command Line

Open your terminal and run:

php index.php email "Hello World"

php index.php sms "Hello via SMS"

If both commands work, your system is now extendable.

Terminology

Extensibility: The ability to add new functionality without modifying existing core logic.

Loose Coupling: Components depend on abstractions, making them easier to replace or extend.
