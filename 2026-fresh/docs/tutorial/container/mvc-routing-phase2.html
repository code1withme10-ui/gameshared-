Phase 2 – Introduce Router Class

Objective

In Phase 1, manual routing used if/else statements inside index.php.

In this phase, we will create a minimal Router.php to cleanly handle GET routes.

Sender.php functionality will remain unchanged.

What Must Change / Key Concepts

  • Create /app/core/Router.php
  • Define GET routes
  • Remove if/else URI logic from index.php
  • Dispatch requests using Router

Example Router class:

<?php // Minimal router class for GET requests class Router { private $routes = []; public function get($uri, $callback) { $this->routes['GET'][$uri] = $callback; } public function dispatch($uri, $method) { $uri = parse_url($uri, PHP_URL_PATH); if(isset($this->routes[$method][$uri])) { $callback = $this->routes[$method][$uri]; $callback(); } else { echo "Page not found."; } } }
This is the simplest Router abstraction.

Terminology

  • Router – A class that maps URLs to callbacks.
  • Dispatch – Execute the callback matching the current URI.
  • Callback – A function or code block called by Router.

Reflection

How does using a Router class improve maintainability over if/else statements?

It centralizes route logic and prepares for Controllers in next phase.
